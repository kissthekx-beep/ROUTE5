# ROUTE5 현재 작업 상태

마지막 정리일: 2026-05-05

## 프로젝트 한 줄 정의

ROUTE5는 혼자인 사람들을 위한 하루 계획표입니다. 사용자의 오늘 기분과 조건을 받아, 그날의 형태에 맞는 하루 루트를 만들어주는 앱입니다.

## 현재 디자인 방향

현재 ROUTE5 디자인은 Studio Gaufrettes 테스트 페이지 느낌을 기준으로 합니다.

- 진한 핑크 배경
- 컬러풀하고 말랑한 찰흙 질감 캐릭터
- 둥둥 떠다니는 부드러운 움직임
- 검은 외곽선은 쓰지 않음
- 선택된 답변은 검은색 칸과 흰색 글자로 표시
- 질문-선택-다음 단계로 이어지는 테스트형 경험

정식 디자인 기준은 `DESIGN.md`에 저장되어 있습니다.

## 1페이지: 감정 수집

질문:

```text
오늘 너의 형태는?
```

설명:

```text
지금의 기분을 하나 골라줘. 선택한 형태에 맞춰 오늘의 하루계획을 만들게.
```

감정 선택지:

- 신나!
- 행복해
- 쓸쓸해
- 우울해
- 싱숭생숭
- 설레임
- 사랑해
- 평화로워
- 외로워
- 따분해
- 화나
- 예민해

버튼:

```text
NEXT
```

역할:

- 아직 루트를 만드는 단계가 아니라 사용자 정보 수집의 첫 단계입니다.
- 기분을 선택하면 선택된 칸이 검은색으로 바뀝니다.
- `NEXT`를 누르면 2페이지로 이동합니다.

## 2페이지: 세부 형태 수집

질문:

```text
좀 더 너의 형태를 알려줘!
```

설명:

```text
숫자보다 감각에 가깝게 골라도 좋아. 오늘의 루트가 네 리듬에 맞춰질 거야.
```

수집 항목:

- 이동수단: 걷는게 좋아, 차로다니는게 좋아, 대중교통이 좋아, 기차가 좋아, 비행기가 좋아
- 이동거리: 1km부터 500km까지 슬라이더
- 함께할사람: 혼자하는게 좋아, 둘이다니고싶어, 셋이상이 좋아
- 비용: 0원부터 100만원까지 슬라이더
- 취미: 직접 입력

취미 입력에서 감지하는 기본 신호:

```text
미술, 음악, 게임, 사진, 컴퓨터, 산책, 강아지, 고양이, 시, 차,
카메라, 잡지, 책, 독서, 음식, 요리, 걷기, 달리기, 러닝, 스포츠
```

버튼:

```text
오늘의 루트 찾기
```

역할:

- 루트를 만들기 위한 조건을 수집합니다.
- 취미 입력 전에는 안내 칩을 보여주지 않습니다.
- 비슷한 취미 단어가 입력되면 검은색 칩으로 감지된 취미가 표시됩니다.

## 3페이지: 루트 결과

목적:

- 1페이지와 2페이지에서 수집한 정보를 바탕으로 사용자에게 맞는 하루 루트를 보여줍니다.

화면 구성:

- 상단에 수집된 정보를 작은 문장으로 요약합니다.
- 그 아래 시간순으로 장소를 보여줍니다.
- 각 장소에는 시간, 장소명, 장소 성격, 한 줄 추천 문장, 장소 사진, 가는길, 리뷰, 예상 비용이 표시됩니다.
- 각 장소 카드 오른쪽의 작은 사진을 클릭하면 큰 이미지 모달이 열립니다.

현재 예시 루트:

```text
11:00 장충단공원 걷기
13:00 약수동 만포막국수 음식
15:00 약수동 페이토갤러리 전시
16:30 약수동 업사이드커피 카페
18:30 CGV 명동역 씨네라이브러리 영화
21:00 약수동 이자카야 쿠시바 마무리
```

하단 버튼:

```text
다른루트 보기 1/5
```

원칙:

- 한 루트는 기본 5곳 이상으로 구성할 수 있습니다.
- 후보 루트는 5개입니다.
- 현재는 프로토타입용 정적 데이터이지만, 기분의 방향, 취미 단어, 이동수단, 거리, 예산에 따라 후보 순서를 다시 정렬합니다.
- +적 기분은 밝고 활동적인 장소를, -적 기분은 조용한 영화/책/산책/전시 같은 회복형 장소를 우선합니다.
- 취미에 `강아지`가 감지되면 강아지 산책/반려견 놀이터 루트가 우선될 수 있습니다.
- 취미에 `스포츠`가 감지되면 러닝/운동/경기장 루트가 우선될 수 있습니다.
- 취미에 `음악`이 감지되면 음악이 좋은 카페, LP 카페, LP바가 들어간 루트가 우선될 수 있습니다.
- 3페이지 하단에는 선택한 이동수단 기준의 미니멀 `가는방법` 지도가 표시됩니다.
- 가는방법의 작은 이동 단계는 `1 → 2` 표기가 아니라 사람 기호와 `120m 도보이동` 같은 거리 중심 문구로 표시됩니다.
- 현재 사진은 임시 대표 이미지와 네이버/구글 이미지 검색 링크를 사용합니다.
- 정식 서비스에서는 슈퍼루터들이 올린 루트북 사진을 우선 노출하는 방식으로 바꿉니다.
- 실제 서비스 단계에서는 네이버/구글 리뷰가 좋은 곳과 실제 위치/영업/거리/예산 정보를 연결해야 합니다.

## 상단 버튼과 메뉴

- 왼쪽 상단 버튼: 메뉴 열기
- 메뉴에는 별도 제목 단어를 표시하지 않습니다.
- 로그인 상태의 메뉴 항목: 루트북보기, 루터북보기, 슈퍼루터북, 오늘의루트짜기, 내정보, 앱 정보 / About, 로그아웃
- 로그아웃 상태의 메뉴 항목: 로그인
- 로그인/로그아웃은 현재 브라우저 저장소를 쓰는 임시 상태 시스템입니다.
- 앱 정보 / About에는 만든 사람 / Creator, 개발 시작일 / Started, 버전 / Version이 한글과 영문으로 표시됩니다.
- 오른쪽 상단 버튼: 뒤로가기
- 2페이지에서 오른쪽 상단 뒤로가기를 누르면 1페이지로 돌아갑니다.

## 현재 구현 파일

- `apps/web/index.html`: 1페이지, 2페이지, 3페이지 화면 구조
- `apps/web/src/main/styles.css`: 핑크 배경, 찰흙 캐릭터, 선택 칸, 슬라이더, 2페이지 UI, 3페이지 루트 결과 스타일
- `apps/web/src/main/app.js`: 감정 선택, NEXT 이동, 2페이지 선택, 슬라이더 값, 취미 신호 감지, 루트 결과 렌더링
- `apps/web/src/features/mood-intake/moods.js`: 감정 목록과 감정별 응답 문구
- `apps/web/src/features/mood-intake/README.md`: 감정 수집 기능 설명
- `apps/web/src/features/routes/route-candidates.js`: 3페이지 루트 후보 데이터
- `apps/web/src/features/routes/README.md`: 루트 후보 데이터 설명
- `DESIGN.md`: 전체 디자인 가이드
- `DESIGM.md`: 오타로 DESIGN.md를 찾을 때를 위한 안내 파일

## 저장된 에셋

- `design/assets/characters/looty-original.png`: 사용자가 제공한 루티 원본 참고 이미지
- `apps/web/public/assets/characters/looty-original.png`: 웹 폴더에 복사된 루티 원본 참고 이미지

현재 첫 화면에서는 루티 원본 이미지를 직접 사용하지 않습니다. 캐릭터는 CSS로 만든 선 없는 찰흙 질감의 컬러 덩어리 캐릭터를 사용합니다.
