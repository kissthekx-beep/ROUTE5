# ROUTE5 디자인 가이드

## 최종 방향

앞으로 ROUTE5의 디자인 기준은 `Looking For Parking`의 픽셀 게임 웹사이트 감성으로 통일합니다.

단, 현재 제품 언어와 색감은 한국 사용자 기준으로 다시 정리합니다. 큰 페이지 제목이나 브랜드성 타이틀은 영어를 사용할 수 있지만, 질문, 선택 문항, 설명, 버튼, 상태 문구, 루트 기록 문구는 한글을 기본으로 합니다. 전체 색감은 흰색을 섞은 파스텔 다홍색과 흰색 글씨를 중심으로 유지하고, 블루는 꼭 필요한 포인트에만 제한적으로 사용합니다.

ROUTE5의 제품 목적:

- ROUTE5의 핵심 목적은 `ROUTIE`라는 캐릭터를 키우는 것입니다.
- ROUTIE를 키우기 위해 사용자는 ROUTE5가 제시하는 하루계획을 완료합니다.
- 하루계획을 완료하면 꾸미기 액세서리 아이템과 우유를 받습니다. 쿠키는 가끔 지급되는 보상입니다.
- 하루계획 5번 완료마다 ROUTIE의 나이가 1살 올라갑니다.
- ROUTIE의 표기는 `LV`가 아니라 나이로 표시합니다. ROUTIE는 처음 시작할 때 1살입니다. 예: `ROUTIE(1살)`.
- 나이 1살의 말은 `루......루....`입니다.
- 나이 2살부터 3살까지의 말은 `루...루티..우..유..`입니다.
- 나이 4살부터 5살까지의 말은 `안녕 루티.. 우유좋아`입니다.
- 나이 5살부터는 ROUTIE가 지역의 맛집이나 갈만한 곳에 대한 힌트를 말할 수 있습니다.
- 나이 10살이 되면 SUPERROUTIE가 되고, SUPERROUTIE 페이지에 들어갈 수 있습니다.
- ROUTIE 나이 아래에는 분홍색 생명바를 둡니다. 기본 시작 생명은 5일이며, 하루가 지날 때마다 줄어듭니다.
- ROUTIE 생명이 3일이면 말풍선에 `배고프다`를 표시하고, 1일이면 `루티 아파요`를 표시합니다.
- ROUTIE 생명이 0일이 되면 그날은 하루계획을 짤 수 없고, ROUTIE ROOM에서 우유나 쿠키를 먹여 회복시킬 수 있습니다. 하루계획은 다음날부터 다시 짤 수 있습니다.
- ROUTIE룸은 루트북과 별개인 ROUTIE 성장 공간입니다.
- ROUTIE ROOM의 영문 표기는 반드시 `ROUTIE ROOM`으로 씁니다.
- 내 ROUTIE룸 제목은 작게 `ROUTIE ROOM`으로 표시하고, 다른 루터의 방은 `아이디의 ROUTIE ROOM` 형식으로 표시합니다.
- ROUTIE룸은 예전 싸이월드 미니홈피처럼 흰색 정사각형 방 중앙에 ROUTIE가 있는 형태로 구성합니다.
- ROUTIE룸의 정사각형 방은 투명한 3D 상자 안에 ROUTIE가 들어가 있는 느낌이어야 합니다. 단, ROUTIE가 들어가 있는 흰색 방 안쪽은 무늬 없이 단색 흰색으로 둡니다.
- ROUTIE룸에는 ROUTIE 이름짓기, ROUTIE 나이, 다음 나이까지 남은 하루계획 수, 아이템가방 버튼을 둡니다.
- ROUTIE 이름짓기는 최초 1회만 가능하며, 이름을 저장한 뒤에는 이름짓기 칸이 화면에서 사라집니다.
- 우유, 쿠키 같은 사용형 아이템은 ROUTIE룸에서만 사용할 수 있습니다.
- 염색약은 랜덤 보상으로 등장할 수 있으며, ROUTIE ROOM에서 사용하면 ROUTIE의 몸 색이 바뀝니다.
- 다른 사용자의 ROUTIE북 또는 루트북에서 아이디를 클릭하면 `ROUTIE룸구경하기`로 진입할 수 있어야 합니다.
- 우유를 먹이면 생명이 3일 회복되고, 쿠키를 먹이면 생명이 5일 회복됩니다.

현재 ROUTE5의 기본 화면 톤:

- 전체 배경은 흰색을 섞은 파스텔톤 다홍색입니다.
- 파란 칸, 파란 패널, 파란 정보 카드는 기본 UI로 사용하지 않습니다.
- 루터북보기의 다른 루터 정보창처럼 `rgba(255, 255, 255, 0.16)` 정도의 투명한 흰색 칸을 모든 칸/패널의 기본값으로 씁니다.
- 파란색 칸은 기본값이 아니며, 정말 필요한 강조 순간에만 파란 구름 모양 칸으로 제한해 사용합니다.
- 정보는 가능하면 칸 없이 흰색 글씨만으로 표시합니다.
- 정말 필요한 강조 영역에만 파란 구름 모양의 칸을 제한적으로 사용합니다.
- 칸과 패널은 필요한 곳에만 쓰고, 중첩된 칸처럼 보이지 않도록 미니멀하게 정리합니다.
- 모든 글씨는 흰색을 기본으로 사용합니다.
- 글자에는 그림자를 넣지 않습니다.
- 첫 화면 배경에는 단순 도형 대신 사람, 꽃, 자동차, 강아지, 고양이, 구름 같은 픽셀 사물 오브젝트를 둥둥 떠다니게 배치합니다.
- 두 번째 페이지 배경도 첫 화면과 같은 픽셀 사물 오브젝트 세트를 사용합니다.
- 세 번째 페이지와 실행 페이지를 포함한 전체 페이지 배경도 이모지가 아니라 같은 픽셀 사물 오브젝트 세트를 사용합니다.
- 전체 배경에는 하단 건물 실루엣을 두지 않고, 다홍색 그리드와 픽셀 사물 오브젝트만 남깁니다.
- 상단 브랜드는 `ROUTE5`만 표시하고, 앞의 숫자 `5` 배지는 사용하지 않습니다.
- 첫 화면의 작은 `ROUTE5 MOOD TEST` 보조 문구는 사용하지 않습니다.
- 첫 화면 다음 버튼은 `NEXT`로 표시합니다.
- 3페이지 `가는방법` 노선에서는 이동거리 칩을 별도 목록으로 만들지 않고, 장소와 장소 사이에 작은 텍스트로 표시합니다. 노선 선은 거리 글자와 겹치지 않게 얇고 미니멀하게 둡니다.
- 3페이지 `가는방법`에는 총 이동거리도 함께 표시합니다.
- 3페이지 `가는방법`에는 첫 장소와 가까운 지하철역, 호선 정보를 함께 표시합니다.
- 이동수단 선택지는 `좋아` 문구를 붙이지 않고 `걷기`, `자동차`, `대중교통`, `기차`, `비행기`처럼 명사형으로 표시합니다.
- 이동거리 슬라이더는 0m-10km 구간이 넓게 조절되도록 배치하고, 10km 이후 장거리 구간은 더 압축해 표시합니다.
- 하루 루트를 완료한 뒤 루트찾기 또는 루트실행 페이지로 다시 들어가려 하면 `오늘의 루트를 끝냈습니다. 내일 루트를 기다려주세요!` 알림을 띄웁니다. 루트북, 루터북, ROUTIE룸으로 돌아가는 뒤로가기는 이 알림을 띄우지 않습니다.
- 실행 페이지 하단 순서는 별점, `오늘 루트 완료`, `내 루트북 보기` 순서로 둡니다.
- 루트북의 `저장된 루트` 영역은 각 줄을 파란 카드로 만들지 않고, 다이어리 종이처럼 다른 색의 한 영역 안에 줄글과 저장 루트가 들어가게 구성합니다.
- 루터북은 사용자가 다른 사람들에게 자신이 저장한 루트 중 괜찮은 루트를 공개하는 공간입니다. 다른 루터의 루티 사진, 루터 아이디, 공개 대표 루트를 보여주고, 아이디를 클릭하면 `루티방구경하기`와 `루트북 구경하기` 선택지를 보여줍니다.
- 루터북에서 다른 루터의 공개 루트를 클릭하면 그 루터가 쓴 코멘트, 가는길, 함께 가기 좋은 팀, 맛집 추천 메뉴, 좋았던 점을 볼 수 있어야 합니다.

이전 ROUTE5 디자인 방향이었던 테스트형 핑크/다홍 배경, 둥근 칩 중심 UI, Streamline Pixel 보조 아이콘 기준은 더 이상 주 방향으로 사용하지 않습니다. 필요한 경우에도 새 픽셀 게임 톤에 맞게 다시 해석합니다.

단, `ROUTIE`는 예외입니다. ROUTIE는 ROUTE5의 성장 동반 캐릭터이므로 픽셀 UI 안에서도 3D 말랑 점토 캐릭터로 유지합니다. ROUTIE의 기준 이미지는 `design/assets/characters/looty-clay-3d-reference.jpg`이며, 웹 구현용 레퍼런스는 `apps/web/public/assets/characters/looty-clay-3d-reference.jpg`에도 둡니다.

ROUTE5는 혼자인 사람들을 위한 하루 루트 앱이지만, 화면 경험은 `웹앱`보다 `작은 픽셀 게임을 플레이하는 느낌`에 가까워야 합니다. 사용자는 기분을 입력하고 루트를 추천받는 사람이 아니라, 도시 안에서 오늘의 루트를 탐색하는 플레이어처럼 느껴야 합니다.

핵심 문장:

```text
ROUTE5는 하루를 고르는 픽셀 도시 게임이다.
```

## 참고 사이트 분석

참고 URL:

```text
https://lookingforparkingthegame.com/
```

확인한 사이트 정보:

- 사이트 제목은 `Looking For Parking`
- 설명은 `A game about finding parking in LA.`
- Phaser 3 기반 웹 게임
- 기준 캔버스는 `480 x 270`
- 게임 월드 폭은 `720`
- `pixelArt: true`, `antialias: false`, `roundPixels: true`
- CSS에서 `image-rendering: pixelated`, `image-rendering: crisp-edges` 사용
- UI 폰트는 `Press Start 2P`
- 기본 배경과 테마 색은 `#000000`
- UI 오버레이는 검정 기반
- 보조 UI 다크 퍼플은 `#3e3b65`
- 텍스트 기본색은 `#eaf4ff`
- 패널은 거의 검정에 가까운 반투명 배경과 옅은 푸른 테두리
- 모바일에서는 가로모드와 풀스크린 진입을 강하게 안내
- 대표 이미지는 LA 거리, 산, 하늘, 야자수, 상점, 주차된 차를 픽셀 아트 레이어로 구성

참고 사이트에서 가져올 핵심:

- 저해상도 픽셀 캔버스의 또렷한 격자감
- Press Start 2P 계열의 게임 UI 타이포
- 검정 오버레이 위 흰색 텍스트와 1px 그림자
- 깜빡이는 안내 문구
- 도시 배경을 여러 레이어로 쌓는 방식
- 도로, 건물, 사람, 차, 간판, 쓰레기통 같은 생활감 있는 오브젝트
- 작은 UI 문구도 게임 명령처럼 보이게 하는 방식
- 화면 전체를 하나의 게임 장면처럼 다루는 구성

그대로 복사하지 않을 것:

- LA와 주차 게임 세계관
- 자동차 주행 플레이 자체
- 특정 상점/랜드마크/캐릭터/차량 이미지
- 원본 로고, 에셋, 음악, 사운드

ROUTE5에 적용할 때는 `주차`가 아니라 `오늘의 루트를 찾아 도시를 이동하는 픽셀 게임`으로 바꿉니다.

## 디자인 원칙

- 모든 시각 요소는 픽셀 격자 위에 놓인 것처럼 보여야 합니다.
- 부드러운 그라데이션, 유리질 카드, 말랑한 클레이 질감, 큰 둥근 캡슐 UI는 주 방향에서 제외합니다. 단, 루티 캐릭터와 루티 아이템은 예외적으로 3D 말랑 점토 입체감을 유지합니다.
- 이미지와 아이콘은 흐릿하게 확대하지 않습니다.
- UI는 8px 단위의 픽셀 리듬을 따릅니다.
- 큰 화면에서도 요소를 무한히 키우지 않고, 작은 게임 캔버스를 확대해 보여주는 감각을 유지합니다.
- 라운드는 최소화합니다. 둥근 모서리가 필요하면 2px-4px 정도만 사용합니다.
- 그림자는 부드러운 블러가 아니라 1px-2px 픽셀 그림자처럼 처리합니다.
- 텍스트는 흰색을 기본으로 하고, 그림자는 붙이지 않습니다.
- 텍스트에는 그림자를 넣지 않습니다. `text-shadow`는 사용하지 않습니다.
- 정보가 많아도 SaaS 대시보드처럼 정리하지 말고, 게임 HUD와 팝업 창처럼 구성합니다.

- 전체 화면을 핑크, 다홍, 보라 단일톤으로 덮는 것
- 부드러운 파스텔 그라데이션
- 루티를 제외한 일반 UI에 클레이 캐릭터용 하이라이트 팔레트를 과하게 쓰는 것
- 유리모피즘식 투명 카드

## 루티 캐릭터 기준

루티는 앞으로 보내준 3D 점토 캐릭터를 기준으로 합니다.

레퍼런스:

```text
design/assets/characters/looty-clay-3d-reference.jpg
apps/web/public/assets/characters/looty-clay-3d-reference.jpg
```

루티의 형태:

- 전체 실루엣은 완전한 원이 아니라 아래가 넓고 위가 부드럽게 솟은 말랑한 물방울/찰흙 덩어리입니다.
- 색은 연노랑과 연두가 섞인 따뜻한 점토색을 기본으로 합니다.
- 얼굴 요소도 검은색 선이나 점이 아니라, 몸과 같은 재질의 돌출 점토 조각이어야 합니다.
- 눈은 몸 표면 위에 붙은 작은 볼록 타원입니다.
- 입은 얇은 선이 아니라 몸 위에 얹힌 둥근 튜브형 미소입니다.
- 왼쪽 아래와 바닥 쪽에는 녹색 기운의 깊은 그림자가 있어야 합니다.
- 오른쪽 위와 중앙 상단에는 부드러운 밝은 하이라이트가 있어야 합니다.
- 픽셀 UI 위에 올라가더라도 루티 자체는 매끈한 3D 입체감을 유지합니다.

루티 렌더링 규칙:

- 앱 화면에서는 임시 CSS 재현보다 사용자가 제공한 레퍼런스 이미지 자체를 우선 사용합니다.
- 현재 웹 프로토타입의 루티 기본 이미지는 `apps/web/public/assets/characters/looty-clay-3d-reference.jpg`를 직접 표시합니다.
- 원본 이미지의 흰 배경이 화면에 네모로 보이지 않도록, 화면에서는 루티 실루엣에 맞춘 마스크/클리핑을 적용합니다.
- 검은 눈, 검은 입, 평면 아이콘식 얼굴은 사용하지 않습니다.
- 윤곽선은 강한 검정선보다 빛/그림자로 입체를 표현합니다.
- CSS로 그릴 때도 `radial-gradient`, `box-shadow`, `filter: drop-shadow`, `inset shadow`를 사용해 점토 덩어리의 부피를 살립니다.
- 이미지로 사용할 때는 흰 배경이 드러나지 않도록 누끼 또는 투명 배경 에셋으로 교체하는 것을 우선합니다.
- 나이가 올라가며 커질 때도 단순 확대만 하지 말고, 같은 입체감과 질감이 유지되어야 합니다.

루티 아이템 기준:

- 아이템도 평면 픽셀 스티커가 아니라 루티 위에 얹힌 작은 3D 점토/소품처럼 보여야 합니다.
- 기본 아이템군은 `리본`, `가발`, `쥬얼리`, `안경`, `눈썹`, `귀걸이`, `모자`입니다.
- 리본은 머리 위나 옆에 얹히되, 부드러운 천/점토 볼륨을 가져야 합니다.
- 가발은 루티의 머리 윗면을 덮는 볼륨감 있는 덩어리로 표현합니다.
- 쥬얼리는 목걸이, 펜던트, 작은 보석 장식처럼 빛나는 입체 포인트로 표현합니다.
- 안경은 얼굴 위에 살짝 떠 있는 3D 프레임처럼 표현합니다.
- 눈썹은 얇은 검정선이 아니라 표면 위에 붙은 둥근 점토 막대처럼 표현합니다.
- 아이템 위치 이동은 ROUTIE룸 안에서만 가능합니다.
- ROUTIE룸 밖에서 배낭 아이템은 착용/벗기만 가능하고 위치 이동은 할 수 없습니다.
- ROUTIE룸 밖에서도 루티에게 조금이라도 걸쳐진 착용 아이템은 잘리지 않고 보여야 합니다.
- 귀걸이는 루티 양옆에 매달린 작은 입체 장식으로 표현합니다.
- 모자는 루티 머리 곡면을 따라 얹힌 둥근 볼륨으로 표현합니다.

## 레이아웃

루트5 화면은 앱 페이지가 아니라 게임 장면처럼 구성합니다.

기본 화면 구조:

```text
검정 또는 도시 픽셀 배경
상단 작은 HUD
중앙 게임 패널 또는 도시 장면
하단 조작/선택 버튼
팝업은 픽셀 박스
```

레이아웃 원칙:

- 화면 중앙에 480 x 270 또는 16:9 비율의 기준 장면을 둡니다.
- 반응형에서는 장면을 crop/contain 방식으로 다루되, 내부 픽셀 리듬을 깨지 않습니다.
- 모바일에서는 세로 전체 스크롤 앱보다, 가로모드 게임 화면 느낌을 우선 검토합니다.
- 카드 목록은 일반 카드 UI가 아니라 픽셀 슬롯, 게임 인벤토리, 스테이지 선택 화면처럼 보이게 합니다.
- 모달은 검정 패널에 밝은 테두리, 픽셀 텍스트, 짧은 선택지로 구성합니다.

## 픽셀 아트 사용

루트5에서 필요한 픽셀 에셋:

- 도시 하늘
- 산 또는 먼 배경
- 서울/동네 실루엣
- 도로 또는 보행로
- 지하철 노선 같은 루트 선
- 장소 카테고리 아이콘
- 작은 사람 캐릭터
- 강아지/고양이 같은 동행 오브젝트
- 카페, 영화관, 서점, 공원, 전시관, 음식점 건물
- 지도 핀, 별점, 사진, 저장, 리뷰 아이콘

픽셀 에셋 원칙:

- 모든 에셋은 낮은 해상도에서 제작하고 CSS/캔버스에서 `image-rendering: pixelated`로 표시합니다.
- 아이콘만 픽셀이고 나머지는 부드러운 UI인 혼합은 피합니다.
- 픽셀 아이콘과 픽셀 배경의 명암 단계가 서로 맞아야 합니다.
- 실제 장소 사진은 픽셀화하지 않아도 되지만, 사진 프레임은 픽셀 UI로 감쌉니다.
- 장소 사진 썸네일은 작은 게임 아이템처럼 배치합니다.

## 애니메이션

참고 사이트는 여러 레이어가 다른 속도로 움직이는 패럴랙스 구조를 사용합니다.

루트5에 적용할 애니메이션:

- 배경 구름이 천천히 이동
- 먼 도시 실루엣은 느리게 이동
- 가까운 도로/사람/간판은 조금 빠르게 이동
- 루트 실행 중 완료된 장소가 픽셀 노선으로 켜짐
- 버튼 선택 시 깜빡임
- 결과 전환 시 화면이 페이드 또는 픽셀 와이프
- 모달은 펼쳐지는 픽셀 박스 느낌

피해야 할 애니메이션:

- 부드러운 3D 회전
- 고무처럼 늘어나는 클레이 모션
- 과한 spring/ease
- 글래스 카드가 떠오르는 모션

## 화면별 방향

### 첫 페이지

목적: 오늘의 기분을 고르는 게임 시작 화면.

디자인:

- 검정 오버레이 위에 픽셀 로고/타이틀
- 작은 도시 배경이 뒤에서 움직임
- `PRESS ANY KEY` 또는 `탭해서 시작` 같은 시작 프롬프트
- 기분 선택지는 스테이지 선택 또는 차량 선택 그리드처럼 구성

### 두 번째 페이지

목적: 이동수단, 거리, 동행, 비용, 취미를 고르는 설정 화면.

디자인:

- 게임 옵션 화면처럼 구성
- 선택된 항목은 밝은 흰색, 비선택 항목은 보라 회색
- 슬라이더는 픽셀 게이지로 변경
- 취미 입력은 게임 이름 입력창처럼 구성

### 세 번째 페이지

목적: 추천 루트 후보를 보여주는 화면.

디자인:

- 루트 후보는 `STAGE 1`, `STAGE 2` 같은 스테이지 카드처럼 구성
- 장소 목록은 시간순 퀘스트 로그처럼 보여줍니다.
- 장소 사진은 작은 픽셀 프레임 안에 둡니다.
- `다른루트 보기`는 다음 스테이지 이동 버튼처럼 만듭니다.
- `이 루트로 실행`은 선택 확정 버튼처럼 만듭니다.

### 실행 페이지

목적: 사진, 한줄평, 별점으로 실제 루트 경험을 기록하는 화면.

디자인:

- 화면 중앙에는 픽셀 도시 지도 또는 미니 노선도를 둡니다.
- 완료된 장소는 픽셀 전구처럼 켜지고 선이 이어집니다.
- 사진 업로드는 작은 아이템 슬롯처럼 보입니다.
- 장소 별점과 루트 별점은 숫자 대신 `★`/`☆` 픽셀 별 기호로 표현합니다.
- 한줄평은 게임 로그 입력창처럼 구성합니다.
- 루트북 저장은 게임 세이브 버튼처럼 보여야 합니다.
- 하단 루트 별점은 한 번 선택하면 잠기고, 별 영역을 다시 누르면 수정 가능한 상태로 돌아옵니다.
- 루트북에 저장할 때는 선택한 루트 별점을 루트 데이터에 함께 붙입니다.

### 루트북 페이지

목적: 완료한 루트 기록을 다시 보고, 좋지 않았던 장소를 루트에서 덜어내는 화면.

디자인:

- 큰 제목은 `ROUTEBOOK`으로 표시합니다.
- 오늘 완료한 루트는 사진, 한줄평, 장소 별점을 장소별 로그처럼 정리합니다.
- 장소별로 `장소 삭제` 버튼을 두어 별로였던 장소를 루트북 기록에서 제거할 수 있게 합니다.
- 하단에는 저장한 루트를 짧고 끌리는 제목으로 모읍니다.
- 저장 루트 100개 이상이면 `슈퍼루트 페이지` 버튼이 열리고, 직접 루트를 짜는 제작 페이지로 넘어가는 흐름을 보여줍니다.
- 루트북도 다홍 배경, 블루 패널, 흰 글씨, 픽셀 사물 오브젝트 톤을 유지합니다.

## 인터랙션

- 선택지는 깜빡이거나 흰색으로 점등됩니다.
- 비활성 선택지는 보라 회색으로 물러납니다.
- 버튼 hover/focus는 블러 그림자가 아니라 픽셀 테두리 또는 1px 이동으로 표현합니다.
- 클릭/선택 후에는 짧게 깜빡이는 confirm beat를 줍니다.
- 모달은 전체 화면을 검정으로 덮고 흰색 픽셀 텍스트로 안내합니다.
- 모바일에서는 작은 버튼을 너무 많이 넣지 말고, 한 줄씩 커다란 픽셀 버튼으로 쌓습니다.

## 루트 추천과 데이터 표현

데이터도 게임처럼 보이게 합니다.

- 별점: 장소와 루트의 경험치
- 사진: 장소 인증 아이템
- 한줄평: 로그
- 루트북 저장: 세이브 파일
- 저장된 루트는 숫자 점수와 별 문자열을 함께 보관합니다. 예: `rating: 5`, `ratingStars: "★★★★★"`.
- 슈퍼루터: 일정 수 이상의 세이브/기록을 가진 고레벨 사용자

슈퍼루터 기준:

```text
루트북 100개 이상 저장한 사용자만 슈퍼루터가 될 수 있다.
```

슈퍼루터는 AI가 만든 루트가 아니라 사람이 직접 짠 루트를 제공할 수 있습니다. 이 기능은 나중에 `루트 제작 모드` 또는 `스테이지 에디터`처럼 설계합니다.

## 구현 규칙

- CSS에는 픽셀 기준 변수를 둡니다.
- 캔버스나 이미지에는 `image-rendering: pixelated`를 사용합니다.
- 큰 UI는 8px 단위로 맞춥니다.
- border-radius는 0, 2px, 4px 정도로 제한합니다.
- box-shadow는 부드러운 블러보다 `1px 1px 0 #000` 같은 픽셀 그림자를 우선합니다.
- 애셋은 `apps/web/public/assets/pixel/` 아래에 둡니다.
- 한글 픽셀 폰트는 `apps/web/public/assets/fonts/` 아래에 둡니다.
- 기존 둥근 캡슐 버튼은 점진적으로 픽셀 버튼으로 교체합니다.
- 기존 다홍/핑크 중심 배경은 검정/남색 픽셀 도시 배경으로 교체합니다.
