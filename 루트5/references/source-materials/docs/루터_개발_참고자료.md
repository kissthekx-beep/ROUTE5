# 루터 앱 개발 참고자료

작성일: 2026-05-04

## 1. 지금 바로 참고할 것: HTML / Netlify 테스트

지금 먼저 필요한 것은 웹에서 바로 열리는 프로토타입이다. 초반에는 Expo보다 **HTML + CSS + JavaScript + Netlify** 조합이 더 편하다.

### Netlify 배포

Netlify는 `index.html` 같은 정적 파일을 드래그해서 바로 배포할 수 있다. 만든 HTML 파일을 `index.html`로 저장한 뒤 Netlify Drop에 올리면 웹주소가 생기는 방식이다.

### 참고 링크

- [Netlify Drop](https://app.netlify.com/drop): HTML 파일을 바로 올려 테스트하는 곳
- [Netlify Docs - Create deploys](https://docs.netlify.com/site-deploys/create-deploys/): 드래그 앤 드롭 배포 설명

### 네 앱에서 쓰는 목적

- 친구나 지인에게 링크 보내서 테스트
- "화면 흐름이 괜찮은지" 확인
- 디자인 수정 전 빠른 피드백 받기

## 2. 앱으로 만들 때 참고할 것: Expo / React Native

나중에 아이폰 앱으로 만들려면 Expo를 쓰는 것이 좋다. Expo는 React Native 앱을 빠르게 만들고 아이폰에서 테스트할 수 있게 해주는 도구다. 공식 문서에서도 `create-expo-app`으로 새 Expo 프로젝트를 만드는 흐름을 안내하고 있다.

### 참고 링크

- [Expo 공식 문서](https://docs.expo.dev/)
- [Create Expo App 문서](https://docs.expo.dev/more/create-expo/)
- [React Native 환경 설정 문서](https://reactnative.dev/docs/set-up-your-environment)

### 네 앱에서 쓰는 목적

- 웹 프로토타입을 실제 아이폰 앱으로 전환
- Expo Go로 아이폰에서 바로 확인
- 나중에 TestFlight / App Store 출시 준비

### 기억할 명령어

```bash
npx create-expo-app router-app
cd router-app
npm start
```

## 3. AI 추천 기능: OpenAI API

앱의 핵심은 **"기분과 예산에 맞춰 하루 루트를 만들어주는 것"**이다. 이걸 진짜 AI 추천처럼 만들려면 OpenAI API를 붙이면 된다. OpenAI 공식 문서에서는 프롬프트로 텍스트를 생성하고, JSON 같은 구조화된 응답을 받는 방식도 설명하고 있다.

### 참고 링크

- [OpenAI API Docs](https://platform.openai.com/docs)
- [Text generation guide](https://platform.openai.com/docs/guides/text)
- [Responses API reference](https://platform.openai.com/docs/api-reference/responses)
- [Structured Outputs guide](https://platform.openai.com/docs/guides/structured-outputs)

### 네 앱에서 쓰는 목적

AI가 해야 할 일은 장소를 "상상"하는 것이 아니라, 아래 역할을 수행하는 것이다.

- 사용자가 선택한 기분 해석
- 예산 조건 반영
- 5개 루트 순서 구성
- 말투를 루터답게 정리
- 루트 설명 생성

### 중요한 설계

AI에게 모든 것을 맡기면 안 된다. 장소, 가격, 지역 데이터는 앱이 가진 데이터에서 고르고, AI는 그 후보를 자연스럽게 조합하게 해야 한다.

### 예시 구조

사용자 입력:

- 기분: 슬픔
- 예산: 10,000원
- 지역: 서울
- 이동수단: 도보

앱 데이터:

- 저렴한 카페
- 무료 산책 장소
- 무료 전시
- 1만원 이하 식당

AI 역할:

- 이 장소들을 자연스러운 하루 흐름으로 정리

## 4. 실제 장소 기반 추천: Google Places API

"현재 있는 맛집과 공간"을 추천하려면 결국 실제 장소 데이터가 필요하다. Google Places API는 장소 검색, 위치 정보, 카테고리, 장소 상세 정보 같은 데이터를 제공하는 서비스다. 공식 문서에서도 장소 검색과 장소 정보 요청 기능을 설명하고 있다.

### 참고 링크

- [Google Places API Overview](https://developers.google.com/maps/documentation/places/web-service/overview)
- [Places API Documentation](https://developers.google.com/maps/documentation/places)
- [Text Search 문서](https://developers.google.com/maps/documentation/places/web-service/text-search)
- [Place Details 문서](https://developers.google.com/maps/documentation/places/web-service/place-details)

### 네 앱에서 쓰는 목적

- 실제 존재하는 카페 / 식당 / 공원 / 전시공간 검색
- 영업 중인지 확인
- 평점 / 주소 / 위치 정보 가져오기
- 나중에 지도 연결

### 하지만 지금 당장은?

처음부터 Google Places API까지 붙이면 복잡해진다. 지금은 서울 장소 100개를 직접 데이터로 관리하고, 앱이 어느 정도 완성되면 Google Places API를 붙이는 것이 좋다.

## 5. 지도 / 길찾기 참고 자료

루터는 단순 장소 추천이 아니라 **"하루 루트"**를 제안하는 앱이기 때문에 지도 기능이 중요해질 수 있다. Google Maps Platform은 Places, Routes, Maps 같은 위치 기반 API를 제공하고, 루트 최적화나 장소 데이터에도 사용할 수 있다.

### 참고 링크

- [Google Maps Platform Documentation](https://developers.google.com/maps/documentation)
- [Routes API Documentation](https://developers.google.com/maps/documentation/routes)
- [Google Maps Platform Routes](https://mapsplatform.google.com/maps-products/routes/)

### 네 앱에서 쓰는 목적

- 장소 간 이동 거리 표시
- 도보 / 대중교통 이동 시간 계산
- 지도 위에 루트 표시
- 너무 먼 장소끼리 묶이지 않게 제한

### 추천 구현 순서

처음부터 지도 화면을 띄우기보다 아래 순서로 가는 것이 좋다.

1. 같은 지역 안에서만 추천
2. 장소 카드에 "네이버 지도에서 보기" 버튼 추가
3. 나중에 지도 화면 추가
4. 마지막에 자동 길찾기 추가

## 7. 앱 핵심 기능 상세 설명

### 1) 감정 선택

사용자는 처음에 오늘의 기분을 선택한다.

예:

- 행복
- 설렘
- 그냥
- 슬픔
- 우울
- 차분함

기분에 따라 루트 분위기가 달라진다.

예:

- 슬픔 / 우울: 조용한 카페, 공원, 전시, 산책
- 행복 / 설렘: 핫플, 쇼핑, 인기 카페, 맛집
- 그냥: 부담 없는 기본 루트

### 2) 예산 선택

예산은 앱 신뢰도에 아주 중요하다.

예:

- 10,000원 이하
- 20,000원 이하
- 30,000원 이하
- 50,000원 이하
- 100,000원 이하

이 조건을 반영해서 고급 식당이 잘못 나오지 않게 해야 한다.

예:

- 10,000원: 광장시장, 김밥, 산책, 무료 전시
- 30,000원: 카페 + 전시 + 적당한 식사
- 100,000원: 파인다이닝, 예약제 공간 가능

### 3) 루트 개수

핵심 조건:

- 하루 루트는 최대 5개까지 추천

여기서 "5개"는 장소 5개라는 뜻으로 잡는 것이 좋다.

예:

1. 11:00 카페
2. 12:30 산책
3. 14:00 전시
4. 16:00 디저트
5. 18:30 식사

### 4) 실제 장소 기반

가짜 장소가 나오면 앱 신뢰도가 바로 떨어진다.

추천 방식은 아래 구조로 가야 한다.

```text
직접 등록한 서울 장소 데이터
+ 가격 정보
+ 지역 정보
+ 감정 태그
+ 카테고리
= 추천 후보
```

그다음 AI는 후보 안에서만 조합한다.

### 5) 동선 설계

동선은 같은 지역 안에서 시작하는 것이 좋다.

예:

- 성수 루트
- 망원 루트
- 한남 루트
- 종로 루트
- 여의도 루트
- 연남 / 합정 루트
- 을지로 / 충무로 루트

처음에는 지역 하나만 돌게 만들고, 나중에 "성수 -> 한남"처럼 2개 지역 연결을 넣으면 된다.

## 8. 참고하면 좋은 경쟁 서비스 / 방향

검색해서 봐야 할 앱이나 서비스는 아래와 같다.

### 1) 데이트팝

데이트 코스 / 놀거리 추천 구조 참고.

### 2) 망고플레이트 / 캐치테이블

맛집 정보 카드, 가격대, 후기 구조 참고.

### 3) 네이버 지도

장소 상세, 리뷰, 길찾기 연결 참고.

### 4) 트리플

여행 일정 카드 UI 참고.

### 5) Google Maps Saved Lists

장소 저장 / 리스트 구조 참고.

여기서 그대로 따라하는 것이 아니라, 네 앱은 **검색이 아니라 "오늘 하루를 대신 짜준다"**는 점을 강조해야 한다.

### 1단계: HTML 프로토타입

기능:

### 2단계: 디자인 정리

목표: 네 스케치처럼 앱 느낌 만들기.

기능:

- 상단바
- 카드형 UI
- 루트 노선도
- 체크리스트
- 루트북

### 3단계: 데이터 확장

목표: 서울 장소 100개를 실제 앱 데이터로 정리.

장소마다 넣을 정보:

```json
{
  "name": "망원한강공원",
  "area": "망원",
  "type": "산책",
  "mood": ["슬픔", "우울", "차분함"],
  "priceLevel": 0,
  "estimatedCost": 0,
  "time": 60,
  "mapKeyword": "망원한강공원"
}
```

### 4단계: AI 연결

목표: 고정 추천이 아니라 매번 자연스러운 루트 생성.

목표: 아이폰에서 실행되는 진짜 앱 만들기.
