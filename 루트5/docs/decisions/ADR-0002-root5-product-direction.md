# ADR-0002: 감정 기반 하루 루트 추천 앱으로 정의

## 상태

채택

## 배경

초기 참고자료와 프로토타입은 루트5를 검색 서비스가 아니라 사용자의 오늘 하루를 대신 구성해주는 서비스로 설명한다. 핵심 조건은 감정, 예산, 이동수단, 실제 장소 데이터, 최대 5개 장소의 하루 루트다.

## 결정

루트5의 MVP는 `감정 선택 -> 조건 입력 -> 루트 후보 5개 추천 -> 루트 실행 -> 루트북 저장` 흐름으로 정의한다.

AI는 실제 장소를 생성하지 않고, 앱이 가진 장소 후보를 자연스러운 하루 루트와 설명으로 조합하는 역할만 맡는다.

## 결과

- 앱 신뢰도는 장소 데이터 품질과 예산/동선 검증에 달린다.
- 하네스는 추천 결과가 실제 장소 데이터 안에서만 나오는지 검증해야 한다.
- 화면 구현은 루트북까지 이어지는 전체 흐름을 먼저 완성한다.
