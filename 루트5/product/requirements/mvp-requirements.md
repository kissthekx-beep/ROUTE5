# 루트5 요구사항

## 기능 요구사항

### FR-001 감정 선택

사용자는 첫 화면에서 오늘의 감정을 선택한다.

초기 감정:

- 그냥
- 행복
- 슬픔
- 설렘
- 우울
- 차분함

### FR-002 세부 조건 입력

사용자는 추천 전 다음 조건을 입력한다.

- 동행: 혼자, 둘, 셋, 넷 이상
- 이동수단: 도보, 자동차, 대중교통, 기차
- 거리: 최소 1km에서 최대 500km
- 예산: 0원에서 100,000원 이상
- 취미 또는 관심사

### FR-003 추천 방식 선택

사용자는 `AI 추천` 또는 `슈퍼루터 추천` 중 하나를 선택한다.

- `AI 추천`: 앱 데이터 후보를 AI가 자연스러운 하루 흐름으로 조합
- `슈퍼루터 추천`: 검증된 큐레이션 또는 커뮤니티 추천 루트

### FR-004 루트 후보 표시

앱은 최대 5개의 후보 루트를 보여준다. 각 후보는 `1/5`처럼 현재 순번을 표시한다.

각 루트는 최대 5개 장소로 구성된다.

각 장소는 다음 정보를 가진다.

- 시간
- 장소명
- 짧은 설명
- 가는 길 링크
- 리뷰 링크

### FR-005 루트 선택과 다른 루트

사용자는 현재 루트를 선택하거나 `다른 루트` 버튼으로 다음 후보를 볼 수 있다.

### FR-006 루트 실행

선택한 루트는 체크리스트 화면으로 전환된다. 사용자는 각 장소 방문 또는 실행 여부를 체크한다.

### FR-007 루트 성공 기록

루트를 완료하면 사용자는 루트북에 기록을 저장한다.

기록 항목:

- 사진
- 루트 제목
- 글
- 실행한 루트
- 날짜

### FR-008 루트북

사용자는 내 루트북을 볼 수 있고, 다른 루터의 루트북을 구경할 수 있다.

### FR-009 루트 공유와 추천

저장한 루트북은 공유하거나 다른 루터에게 추천할 수 있다.

## 비기능 요구사항

- 실제 장소 데이터 기반이어야 한다.
- 예산을 초과하는 추천은 피한다.
- MVP에서는 같은 지역 안의 장소를 우선 묶는다.
- 추천 실패 시 조건을 완화하라는 안내를 제공한다.
- 개인정보와 위치 정보는 최소한으로 다룬다.
- 웹 프로토타입은 모바일 화면 폭을 기준으로 만든다.
